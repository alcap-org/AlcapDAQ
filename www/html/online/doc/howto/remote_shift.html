Remote shift howto

The job of the remote shift taker is to make sure that the experiment doesn't "catch fire" while the local crew takes a break from constant shift taking. That means the remote shift taker monitors the experiment, fixes minor problems, records experimental changes, and contacts someone local if a problem that disrupts data taking occurs. This e-log will outline how to perform these duties.

Contact phone number: 011 41 56 310 5883

First, because of the PSI firewall an SSH tunnel with several port forwards are necessary. The command I use from linux is:
ssh mulan@mulanbe.psi.ch -L 8000:mulanbe:80 -L 8001:be02:8081 -L 8002:mulanbe:8080 -L 8003:be02:8080 -L 5900:sepii:5900
Similar port forwarding can be accomplished with any ssh program such as putty. For information on how to port forward with putty see (web page needed)
Several links in this e-log assume you use the same ports as me.
Here is how those five ports are used:
  • 8000:mulanbe:80 gives access to the online histograms and run log pages
    • http://localhost:8000/online (online analysis histograms, trend plots, etc)
    • http://localhost:8000/mulan (click "Run Log" or "Last 20 Runs" to get to run log)
  • 8001:be02:8081 gives access to midas web page
    • http://localhost:8001
  • 8002:mulanbe:8080 gives access to elog at PSI, this is necesary to make posts.
    • http://localhost:8002/Summer2006
  • 8003:be02:8081 gives access to raid status page
    • https://localhost:8003 (note the https)
  • 5900:sepii:5900 gives access to the separator vnc server
    • vncviewer localhost:0 (you need the vnc password)
Another useful webpage is the PSI accelerator status http://www.acs.psi.ch/acs/ACSstatus-g-r.html

What to monitor:
  • The most important thing to watch is that the Event Builder data rate is what you expect (2-4 MB/s).
    This is found on the midas status page near the top. If it suddenly drops (or increases) this could indicate several
    possibilities:
    • Proton current has changed to or from "cosmic mode" (check proton status page)
    • Something is wrong with the daq (are all expected frontends running? see elog:234)
    • Separator has tripped (check vnc session or click SeparatorMonitor on midas status page, values are: HV current, vacuum, and status)
    • The kicker has tripped (call the contact number above).
  • Next, watch several subsystems and plots to make sure things are running correctly:
    • Make sure the online analyzer is running. The right most column on the midas status page should be green. If not go to the programs page of midas and click "Start Analyzer".
    • Check that the lazy tape program(s) are running. The lazy label should be as described in elog:504
    • Monitor the online histograms and trend plots. If problems are observed on these plots usually just note the problem, but call the number above if significant problems are suspected.
      • WFD raw time and WFD raw fill
      • Slow Control->VME monitor and Slow Control->Beamline
      • Trend Plots->tau,N/B
  • If conditions change (such as proton beam stops) be sure to change the runlog by going to /Experiment/Edit on Start and changing the correct parameters.
    Also you can retroactively change the runlog by going to the run log page.

Things to do at the end of the shift:
  • Check the status of the raid (https://localhost:8003/) log in as administrator using mulan "root" password.
  • Make sure the local shift people have arrived (we should use IM or something to make sure people are present).
  • Write a shift summary e-log including any comments on problems or oddities.
